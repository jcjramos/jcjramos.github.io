Telco ML

September 6, 2019

While having some training on machine learning, I’ve been thinking in how it could be applied to telco software.

There is a huge array of equipment necessary to enable you to use your mobile phone with minimal issues, your operator has lots of radio equipment spread around that enables your phone to communicate with the network and a core network that enables the data to travel around ( this is of course a huge simplification ). Lot’s of equipment spread in larger areas means of course lot’s of problems, there are the obvious equipment malfunctions, there are power breakdowns and even more “unexpected” events like someone stealing a network antenna. Telco operators employ lot’s of people and use sophisticated software to ensure that their systems are functional, this is a growing challenge as more and more equipment is necessary to support newer network standards like 5G. Several functions of the Telco OSS systems, OSS meaning Operations Support System not Open Source Software, involve something like :

  1. There’s network data coming in (a) ->
  2. Data is analysed by a pre-programmed set of rules ->
  3. Automated corrective actions are performed (b) ->
  4. Problem Tickets are created for the problem to be accessed manually

Looking at this workflow, it is not so hard to think how AI, in particular machine learning can be used to improve existing systems. One of the areas of Machine Learning is Supervised Learning, it is used in most of the practical implementations of machine learning these days and it basically consists on an algorithm that can be trained and corrected by a teacher.

A “training” system could consist in something like :

Stored network historical data + Stored historical problems = Automatic generated rules

And then a live AI based system :
1. network data coming in ->
2. Data is analysed by automatic generated rules ->
3. Automated corrective actions are performed ( c ) ->
4. Problem Tickets are created for the problem to be accessed manually

Notes:
( a ) Including Performance data and fault data generated from network elements
( b ) Tipically simple actions like a Reboot
( c ) As these could be “trained” as well the ratio of problems that need manual human intervention should be reduced as well


I guess most Telcos and respective software suppliers are think of this; but there are a number of challenges to overcome in order to have a functional system – will try to talk about them in a future post. Anyway whoever does this right might revolutionize telco software.
